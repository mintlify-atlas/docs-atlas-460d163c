Once you have `rafnixg` installed, you can either run the CLI command for an instant terminal display or import the library's classes into your own Python scripts. This guide walks through both approaches.

<Tabs>
  <Tab title="CLI">
    The fastest way to use `rafnixg` is the command-line interface. A single command renders your personal card, curated links, and latest blog posts — all formatted with Rich tables.

    <Steps>
      <Step title="Run the CLI command">
        After installing the package, run:

        ```bash
        rafnixg
        ```

        This executes the `main()` function, which does three things in sequence:

        1. Calls `get_links()` — displays a table of your personal links.
        2. Calls `posts()` — fetches and displays your latest Hashnode blog posts.
        3. Calls `display()` — renders your full personal card with name, position, social handles, and bio.
      </Step>

      <Step title="Review the output">
        You will see three Rich-formatted tables printed to your terminal, each with a bold magenta title and highlighted rows. No configuration or API keys are needed to get started.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Python API">
    You can import any of the three main classes directly into your Python scripts for programmatic access.

    <Steps>
      <Step title="Display your personal card">
        The `RafnixG` class renders your profile as a styled terminal table:

        ```python
        from rafnixg import RafnixG

        me = RafnixG()
        me.display()
        ```

        You can also call individual methods:

        ```python
        me.get_links()  # display personal links table
        me.posts()      # display latest blog posts table
        me.display()    # display full personal card table
        ```
      </Step>

      <Step title="Fetch blog posts">
        The `BlogPosts` class queries the Hashnode GraphQL API and returns a list of `BlogPost` objects:

        ```python
        from rafnixg import BlogPosts

        blog = BlogPosts()
        posts = blog.get_posts()

        for post in posts:
            print(post.title, post.link)
        ```

        Each `BlogPost` object has a `title` and a `link` attribute you can use in your own output or tooling.
      </Step>

      <Step title="Retrieve resume data">
        The `Resume` class fetches a hosted JSON Resume file and deserializes it into typed Python objects:

        ```python
        from rafnixg import Resume

        resume = Resume()
        data = resume.get_resume()
        print(data)
        ```

        The returned object contains structured sections including `Basics`, `Work`, `Education`, `Skill`, `Language`, `Certificate`, `Award`, `Project`, and `Reference`.
      </Step>

      <Step title="Import everything at once">
        All three classes are available from the top-level package:

        ```python
        from rafnixg import RafnixG, BlogPosts, Resume

        me = RafnixG()
        me.display()

        blog = BlogPosts()
        posts = blog.get_posts()
        for post in posts:
            print(post.title, post.link)

        resume = Resume()
        data = resume.get_resume()
        print(data)
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>
